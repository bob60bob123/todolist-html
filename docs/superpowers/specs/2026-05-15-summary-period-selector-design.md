# 工作总结周期选择器优化 - 设计文档

**日期**: 2026-05-15
**类型**: 功能优化

## 背景

当前工作总结模块的周期选择器存在以下问题：
1. "过去一周"功能使用率低，需要删除
2. "本周/本月/本季/全年"为固定周期，无法满足灵活选择需求

## 目标

优化工作总结的周期选择器，提供更灵活的周度、月度、季度、年度选择功能。

## 界面变化

### 保留的选项（4个固定快捷选项）
- 过去1日
- 过去2日
- 过去3日
- 今日

### 替换的选项
- 原"本周、本月、本季、全年" → 合并为一个"📅 周期选择"按钮

### 交互方式
点击"📅 周期选择"按钮，展开下拉面板，默认显示上次使用的周期类型选项卡。

## 各周期选项详情

### 1. 周度选择
- **选项范围**: 本年内所有周（约52周）
- **分组方式**: 按月分组，每组4-5周
- **显示格式**: "YYYY年 第W周 (MM/DD - MM/DD)"
- **示例**: "2026年 第20周 (05/11 - 05/17)"

### 2. 月度选择
- **选项范围**: 最近12个月
- **分组方式**: 按年分组（今年月份 + 去年月份）
- **显示格式**: "YYYY年MM月"
- **示例**: "2026年05月"

### 3. 季度选择
- **选项范围**: 最近4个季度
- **分组方式**: 按年分组
- **显示格式**: "YYYY年第Q季度"
- **示例**: "2026年第2季度"

### 4. 年度选择
- **选项范围**: 最近5年
- **显示格式**: "YYYY年"
- **示例**: "2026年"

## 交互流程

1. 点击"📅 周期选择"按钮 → 展开下拉面板
2. 面板显示四个选项卡：周度 | 月度 | 季度 | 年度
3. 点击选项卡切换类型，面板内容随类型变化
4. 点击快捷选项 → 立即应用并关闭面板
5. 按钮更新显示当前选择的周期信息

## 去掉的周期选项
- 过去一周（pastweek）- 从周期选择器中移除

## 兼容性
- 保持 localStorage 数据结构不变
- 已完成的事项使用 completedAt 字段进行周期过滤
